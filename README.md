# Единый портал поддержки бизнеса — Байтерек

> Решение хакатона по разработке информационной системы «Единый портал поддержки бизнеса» для АО «НИХ «Байтерек»

---

## О проекте

**Байтерек** — единая цифровая платформа для предпринимателей Казахстана, объединяющая все меры государственной поддержки от семи дочерних организаций холдинга в одном окне.

До создания портала предприниматели были вынуждены:
- Посещать 7 разных сайтов дочерних организаций
- Самостоятельно разбираться в 70+ мерах поддержки
- Подавать заявки по разным процессам в каждой организации
- Не иметь единого статуса своих обращений

**Наше решение** закрывает все эти боли: один портал — всё государственное финансирование, гарантии, лизинг, экспортная поддержка и консультации.

## Технологический стек

### Frontend
- **React 19** + React Router v6
- **Tailwind CSS v4** — дизайн-система с кастомными токенами (`--color-accent`, `--shadow-card` и др.)
- **Framer Motion v12** — анимации переходов, hover-эффекты, AnimatePresence
- **Zustand v5** — стейт-менеджмент (auth, services, news, notifications, bookings)
- **lucide-react** — иконки (никаких эмодзи в UI)
- **CRACO** — кастомный конфиг webpack/PostCSS поверх Create React App

### Backend
- **Flask 3.0** — REST API
- **Flask-JWT-Extended** — авторизация (access 24h + refresh 30d)
- **Flask-SQLAlchemy** + **Flask-Migrate** (Alembic) — ORM и миграции
- **Flask-Marshmallow** — сериализация/десериализация
- **SQLite** (разработка) / **PostgreSQL** (прод)
- **pytest** + pytest-flask — покрытие тестами

### Инфраструктура
- Docker + docker-compose
- CORS настроен для localhost:3000/3001
- Интеграция с ЕИШ (единая интеграционная шина) и ЕЦП (электронная цифровая подпись)

## Быстрый старт

### Требования
- Python 3.11+
- Node.js 18+
- npm 9+

---

### 1. Клонировать репозиторий

```bash
git clone <repo-url>
cd Baiterek
```

---

### 2. Запустить бэкенд

```bash
cd backend

# Создать виртуальное окружение
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# Установить зависимости
pip install -r requirements.txt

# Инициализировать БД и применить миграции
flask db upgrade

# Заполнить БД тестовыми данными (опционально)
flask shell -c "from app.seeder import seed; seed()"

# Запустить сервер
flask run --port=1489
```

Бэкенд будет доступен по адресу: `http://localhost:1489`

Проверка: `curl http://localhost:1489/api/health`

---

### 3. Запустить фронтенд

```bash
cd frontend

# Установить зависимости
npm install

# Запустить dev-сервер
npm start
```

Фронтенд будет доступен по адресу: `http://localhost:3000`

> API-запросы проксируются на `http://localhost:1489` через переменную окружения `REACT_APP_API_URL`.

---

### 4. Docker (опционально)

```bash
cd backend
docker-compose up --build
```

## API — основные эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/auth/register` | Регистрация |
| POST | `/api/auth/login` | Вход (возвращает JWT) |
| GET | `/api/auth/profile` | Профиль текущего пользователя |
| GET | `/api/services` | Список услуг (фильтры: category, subsidiary, q) |
| GET | `/api/services/:slug` | Карточка услуги |
| GET | `/api/news` | Новости (фильтр: type) |
| GET | `/api/articles` | База знаний (фильтр: category) |
| GET | `/api/subsidiaries` | Дочерние организации |
| POST | `/api/applications` | Создать заявку |
| GET | `/api/applications` | Мои заявки (требует JWT) |
| GET | `/api/notifications` | Уведомления (требует JWT) |
| POST | `/api/bookings` | Создать бронирование |
| GET | `/api/search?q=` | Полнотекстовый поиск |
| GET | `/api/health` | Health-check |

## Почему наше решение

### Закрываем все требования ТЗ

- **Принцип единого окна** — один портал для всех 7 дочерних организаций и 70+ услуг
- **Личный кабинет** — подача заявок, статусы, документооборот, уведомления
- **База знаний** — справочные материалы и инструкции для предпринимателей
- **Калькуляторы** — инструменты для предварительного расчёта параметров финансирования
- **Электронная запись** — онлайн-бронирование консультаций в офисах организаций
- **Административная панель** — управление контентом, заявками, пользователями
- **Интеграционная готовность** — предусмотрены точки интеграции с ЕИШ и ЕЦП

### Техническое превосходство

- **Производительность** — Tailwind CSS v4 генерирует минимальный CSS (~6 КБ gzip), Zustand не создаёт лишних ре-рендеров
- **UX без компромиссов** — анимации на 60fps через Framer Motion, skeleton-загрузка вместо спиннеров, оптимистичные обновления
- **Отказоустойчивость** — при недоступном бэкенде фронтенд корректно откатывается на mock-данные
- **Доступность** — семантическая разметка, ARIA-метки, работа с клавиатурой
- **Безопасность** — JWT с ротацией токенов, CORS whitelist, валидация на всех слоях
- **Масштабируемость** — blueprint-архитектура Flask, Zustand-сторы легко расширяются

### Качество кода

- TypeScript-ready структура с чёткими контрактами API
- Единая дизайн-система на CSS-токенах (один файл `index.css` правит весь визуал)
- Нет дублирования — общие компоненты (`Badge`, `Button`, `Icon`, `Skeleton`) переиспользуются везде
- Тесты на критичные эндпоинты (auth, applications, documents)
